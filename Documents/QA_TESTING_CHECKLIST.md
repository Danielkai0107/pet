# 上線前測試 QC 清單

## 📋 測試概述

本文件提供完整的上線前測試清單，確保系統穩定性、功能完整性和用戶體驗品質，達到正式販售標準。

**測試環境要求**：

- ✅ Staging 環境（Firebase 測試專案）
- ✅ 測試用 LINE 官方帳號
- ✅ 測試用商家帳號（至少 2 個）
- ✅ iOS 和 Android 真機測試

---

## 🎯 測試優先級分類

- **P0 (Critical)**：核心功能，必須通過才能上線
- **P1 (High)**：重要功能，影響用戶體驗
- **P2 (Medium)**：一般功能，可容許輕微問題
- **P3 (Low)**：錦上添花，不影響核心流程

---

## 一、客戶端功能測試（LINE LIFF）

### 1.1 首頁功能 [P0]

#### 測試項目：LINE 登入

- [ ] **LIFF 初始化成功**

  - 開啟 LIFF URL，檢查 Loading 畫面

- [ ] **用戶資料正確顯示**

  - 顯示 LINE 頭像

  - 顯示已儲存的手機號碼（如有）

- [ ] **商家 Logo 顯示**
  - 有設定 Logo 時正確顯示
  - 無 Logo 時顯示預設圖示
  - 圖片載入失敗時的處理

#### 測試項目：個人資料編輯 [P0]

- [ ] **編輯功能**

  - 點擊編輯按鈕開啟編輯模式
  - 姓名可以修改並儲存
  - 手機號碼格式驗證（09XXXXXXXX）
  - 儲存成功後顯示提示訊息
  - 取消編輯後資料不變

- [ ] **錯誤處理**
  - 手機號碼格式錯誤時顯示錯誤提示
  - 空白姓名時無法儲存
  - 網路錯誤時顯示友善訊息

#### 測試項目：服務歷史顯示 [P1]

- [ ] 顯示最近完成的預約記錄
- [ ] 顯示服務日期、寵物名稱、服務項目

#### 測試項目：步驟二 - 寵物資訊 [P0]

- [ ] **必填欄位驗證**

  - 寵物名稱（必填）

- [ ] **選填欄位**

  - 寵物品種（可填可不填）

#### 測試項目：步驟三 - 確認預約 [P0]

- [ ] **預約摘要顯示**

  - 顯示選擇的服務項目、價格、時長
  - 顯示預約日期與時間
  - 顯示寵物資訊
  - 顯示客戶姓名與手機

- [ ] **提交預約**

  - 提交成功顯示成功訊息
  - Loading 狀態顯示
  - 時間衝突時顯示錯誤提示（Transaction 機制）
  - 網路錯誤時顯示友善訊息
  - 成功後可選擇關閉視窗

- [ ] **LINE 通知發送**
  - 預約成功後自動發送 LINE 訊息
  - Flex Message 格式正確
  - 訊息包含預約詳情
  - 訊息包含「查看預約詳情」按鈕
  - 按鈕連結正確（回到 LIFF）

#### 測試項目：邊界情況測試 [P0]

- [ ] **同時預約衝突**

  - 兩個客戶同時預約同一時段
  - 只有一個成功，另一個顯示時段已滿
  - Transaction 機制正確運作

- [ ] **弱網路環境**
  - 3G 網路環境下完成預約流程
  - 上傳照片時間合理（< 10 秒）
