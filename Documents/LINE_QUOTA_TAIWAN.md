## 台灣地區方案

### 輕用量方案（免費）

- **月費**：0 元
- **免費訊息則數**：200 則
- **適用對象**：小型店家、初期測試

### 中用量方案

- **月費**：800 元
- **免費訊息則數**：3,000 則
- **特別注意**：**不可加購訊息**
  - 用完免費訊息後，如需更多訊息
  - 必須先升級至高用量方案
  - 無法單獨購買額外訊息

### 高用量方案

- **月費**：1,200 元
- **免費訊息則數**：6,000 則
- **加購訊息**：可加購
  - 每則從 0.2 元起
  - 採階梯式累進計價
  - 訊息量越多，單價越低

## 計入配額的訊息類型

✅ **會計入配額**（Push API）：

- 預約通知
- 完成通知
- 臨時回報
- 婉拒通知
- 群發訊息

❌ **不計入配額**（Reply API）：

- 自動回覆
- 聊天室即時回覆
- AI 回覆

---

## 系統如何計算

### 官方配額（LINE API）

系統透過 LINE Messaging API 自動查詢：

```
GET /v2/bot/message/quota          // 取得配額上限
GET /v2/bot/message/quota/consumption  // 取得已使用數量
```

**優點**：

- 即時同步 LINE 官方數據
- 準確反映剩餘可用數量
- 自動適應方案變更

### 系統統計（Firestore）

系統內部記錄的發送數量：

- 儲存在 Firestore `messageStats` collection
- 按月統計，按類型分類
- 作為參考和驗證用途

**可能差異**：

- 系統統計可能略多於官方配額
- 因為包含了所有發送嘗試
- 以官方配額為準

## 配額管理建議

### 輕用量方案（200 則/月）

**適合情境**：

- 新開業店家
- 預約量較少（每天 < 7 則訊息）
- 主要使用自動回覆功能

**使用策略**：

1. 優先使用自動回覆（不計入配額）
2. Push 訊息保留給重要通知
3. 監控每日使用量（約 6-7 則/日）
4. 達到 80% 時考慮升級

**範例計算**：

- 每天 3 個預約 × 2 則通知（確認 + 完成） = 6 則/日
- 30 天 = 180 則/月
- 剩餘 20 則作為緩衝

### 中用量方案（3,000 則/月）

**適合情境**：

- 成長中店家
- 預約量中等（每天 10-30 個預約）
- 需要發送提醒和行銷訊息

**使用策略**：

1. 每天約 100 則配額
2. 可額外發送提醒訊息
3. 注意：無法加購，超過需升級

### 高用量方案（6,000 則/月 + 加購）

**適合情境**：

- 大型店家或連鎖店
- 高預約量（每天 > 50 個預約）
- 需要群發行銷訊息

**使用策略**：

1. 充足的基本配額
2. 超過時可加購（0.2 元/則起）
3. 適合穩定營運的店家

## 升級時機判斷

### 何時該從輕用量升級到中用量？

✅ **建議升級**：

- 連續 3 個月達到 80% 以上使用率
- 每月超過 160 則
- 經常需要延遲發送訊息
- 想加入提醒和行銷功能

❌ **可以維持輕用量**：

- 每月使用量 < 150 則
- 主要依賴自動回覆
- 預約量穩定且較少

### 何時該從中用量升級到高用量？

✅ **建議升級**：

- 每月使用量 > 2,500 則
- 想要加購更多訊息
- 需要彈性的配額管理
- 有群發行銷需求

## 成本效益分析

### 輕用量 vs 中用量

**輕用量**：

- 成本：0 元
- 訊息：200 則
- 單則成本：0 元

**中用量**：

- 成本：800 元
- 訊息：3,000 則
- 單則成本：0.27 元

**建議**：

- 當訊息需求 > 200 則時，中用量方案划算
- 即使只用 500 則，平均 1.6 元/則，仍比輕用量受限好

### 中用量 vs 高用量

**中用量**：

- 成本：800 元
- 訊息：3,000 則
- 限制：不可加購

**高用量**：

- 成本：1,200 元（基本）
- 訊息：6,000 則（基本）
- 彈性：可加購（0.2 元/則起）

**建議**：

- 如果穩定需要 > 3,000 則，選高用量
- 需要彈性加購，選高用量
- 預算有限且用量 < 3,000，選中用量

## 常見問題

### Q1: 為什麼系統統計和 LINE 官方配額不一樣？

**A**: 這是正常的：

- LINE 官方配額：只計算成功的 Push API 訊息
- 系統統計：包含所有發送記錄和嘗試
- 以 LINE 官方配額為準

### Q2: 自動回覆會用掉配額嗎？

**A**: 不會！

- 自動回覆使用 Reply API
- Reply API 的訊息完全不計入配額
- 可以無限制使用自動回覆功能

### Q3: 超過 200 則會發生什麼事？

**A**:

- 無法繼續發送 Push 訊息
- 自動回覆仍可正常運作
- 需要等到下個月或升級方案

### Q4: 如何知道我該用哪個方案？

**A**: 參考您的月均使用量：

- < 150 則/月 → 輕用量（免費）
- 150-2,500 則/月 → 中用量（800 元）
- > 2,500 則/月 → 高用量（1,200 元）

### Q5: 可以隨時升級或降級嗎？

**A**:

- 升級：隨時可以，立即生效
- 降級：需要在官方帳號管理後台操作
- 建議在月初調整，避免浪費費用
